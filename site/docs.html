Build your own offline tldraw desktop app.

Quick start

deno task desktop:dev

Use Deno 2.9 or newer. For a browser-only Vite dev server, run `deno task dev`.

Architecture

`desktop.ts` is the Deno Desktop entry. It serves the built Vite output from `dist/`, configures the native application menu, and routes menu clicks into the webview with `executeJs`.

`src/App.tsx` renders `` and installs a small `globalThis.tldrawDesktop` bridge that exposes file and edit actions to the native menu layer.

Native menus

Edit commands call tldraw's built-in UI action IDs: undo, redo, cut, copy, paste, and select-all. This avoids relying on generic OS edit roles that only target focused text fields.

Local files

Open loads `.json` or `.tldr.json` snapshots. Save exports `editor.getSnapshot()` as a `.tldr.json` file. When the File System Access API is available, Save writes back to the current file and Save As opens a save picker. This keeps the template local-first without requiring a sync server.

Compatibility

Chromium-based webviews expose browser-native file open and save pickers through the File System Access API. If that API is not available or a picker call is blocked, the template falls back to file input and download behavior.

The template uses the CEF backend because the default Windows WebView backend crashed during package smoke testing on this tldraw canvas.
